# 提交策略与确认制度

## 强制确认

任何真正消耗天池提交次数、论坛发帖、上传作品、提交文件、发送评审邮件、发布 Kaggle Dataset、发布 Kaggle Notebook 的动作，都必须先列出以下信息并等待用户明确确认：

1. 题目 ID。
2. 题目名称。
3. 提交目标页面或平台。
4. 提交文件路径。
5. 文件大小。
6. 生成命令。
7. 今日剩余提交次数或总剩余次数。
8. 预计消耗的机会。
9. 预期影响和风险。

未收到用户明确确认前，只能做本地校验、页面查看和草稿准备。

## 提交前本地校验

所有赛道都必须执行：

1. 行数或文件数量正确。
2. 列名或目录结构正确。
3. ID 完整且不重复。
4. 没有空值、非法值或编码错误。
5. 压缩格式正确。
6. 文件名符合官方要求。
7. 结果可由代码和配置复现。

## 提交后记录

每次提交后必须写入：

- `experiments.csv`
- `reports/submission_log.md`

记录字段：

- 提交时间。
- 文件路径。
- 赛道。
- public score。
- 排名。
- 页面反馈。
- 是否成功。
- 复现命令。

## 禁止事项

1. 不删除原始数据。
2. 不覆盖历史提交。
3. 不把账号、cookie、token 写入文档。
4. 未确认规则时不使用外部数据或闭源 API。
5. 未确认前不消耗提交次数。
6. 遇到验证码、登录、实名认证、报名、队伍确认、提交确认时暂停。
