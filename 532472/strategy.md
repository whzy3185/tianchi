# 策略文档

## 1. 题目理解

大模型知识编辑任务。目标是在指定事实上更新模型知识，同时尽量保持泛化、可迁移性、局部性和语言流畅性。

## 2. 数据类型判断

LLM 指令/知识编辑 JSON 数据，核心是 prompt、target_new、case_id 和评测字段。

## 3. 评测指标解释

官方返回综合 `score`，并包含 Edit_Success、Generalization、Portability、Locality、fluency 等子指标。策略上不能只追求编辑成功，还要避免破坏无关知识。

## 4. 最小 baseline

优先跑官方 EasyEdit 示例，使用最小样本完成数据读取、编辑、预测、提交 JSON 生成和校验。

## 5. 进阶方案

- 比较 EasyEdit 内置编辑方法和超参数。
- 针对不同知识类型设置不同编辑强度。
- 保留原始输出和编辑后输出用于本地回归检查。

## 6. 特征工程方向

本题不是传统特征工程。重点是样本分组、prompt 模板、编辑层/位置、编辑强度、生成参数。

## 7. 模型选择

优先使用官方 baseline 推荐模型和 EasyEdit 支持模型。GPU 不足时先跑小模型/少量样本 smoke test。

## 8. 验证方案

本地验证 JSON 结构、样本覆盖、case_id 唯一性；若有官方评测脚本，按官方脚本计算子指标。

## 9. 后处理方案

仅做格式清洗和空输出兜底，不做违反规则的测试集标注或相似数据构造。

## 10. 融合方案

可按样本类型选择不同编辑方法，但必须保证复现简单，提交前固定配置。

## 11. 提交计划

1. 下载并确认 PersonUKE 数据。
2. 跑通 EasyEdit baseline。
3. 先提交一次格式正确结果。
4. 根据线上子指标调参。

## 12. 风险点

GPU/模型环境重；训练字段限制严格；Top3 需要代码和模型权重复现。

## 13. 时间优先级

高。已报名且初赛截止较近。

## 14. 是否建议深做

建议深做，但应排在 532474 之后。
