# 背景文档

## 1. 赛题名称

CCKS2026-任务三：大模型知识编辑评测

## 2. 赛题链接

https://tianchi.aliyun.com/competition/entrance/532472

## 3. 赛事背景

本任务聚焦大模型知识编辑。大模型将知识存储在参数空间中，存在知识更新成本高、知识滞后、事实错误和全量微调成本高等问题。知识编辑希望在不重新训练完整模型的前提下，对指定知识进行更新、修改或删除，并保持模型其他能力稳定。

## 4. 任务目标

对大模型执行知识编辑，使模型在目标知识上输出更新后的事实，同时在泛化、可迁移性、局部性和流畅性方面保持良好表现。

## 5. 输入数据说明

官方说明数据集为 PersonUKE。参赛方法在训练时只允许使用 edit/train prompt 和 `target_new` 字段；其他字段仅用于评估。

## 6. 输出结果说明

最终提交一个 JSON 文件，命名为 `参赛队伍名称_result.json`。每个样本需包含 `case_id`、`requested_rewrite`、`pre`、`post` 等评测所需结构，具体字段以官方样例和提交页为准。

## 7. 评测指标

官方页面列出的核心指标包括：

- Edit_Success
- Generalization
- Portability
- Locality
- fluency
- 综合 `score`

具体权重待官方评测脚本确认。

## 8. 参赛限制

- 每个团队 2-3 人。
- 每位选手只能加入一支队伍。
- 报名、组队变更、实名认证截止：2026-07-15 23:59:59。
- 复赛前 3 名需提交代码和编辑后模型 checkpoint 进行复现审核。

## 9. 是否允许外部数据

允许使用公开可获得的预训练数据，如词向量、字向量等。严禁标注测试集或构造测试集相似数据进行训练。

## 10. 是否允许预训练模型

待确认。官方允许开源代码或工具，并提供 EasyEdit 作为 baseline；实际可用模型范围以赛题页和论坛公告为准。

## 11. 是否允许在线 API

待确认。不要默认使用闭源在线 API 生成结果，除非官方公告明确允许。

## 12. 是否需要提交代码、模型、报告

- 常规提交：提交预测 JSON 文件。

- 论文/报告：评测论文时间为 2026-09-01，具体要求待 CCKS 2026 通知。
